# Тестер простых matplotlib-программ в ejudge

Программа предназначена для тестирования простых программ построения
диаграмм с использованием библиотеки `matplotlib`.

По мотивам проекта\
https://github.com/ShashkovS/matplotlib_tester

Программа пользователя **не рисует график**, а вместо этого формирует
**текстовое описание диаграммы**, которое удобно анализировать.

Часто для проверки достаточно **сравнить вывод пользовательской и
эталонной программы**.

------------------------------------------------------------------------

# Основная идея

Тестер **не использует настоящую библиотеку `matplotlib`**.

Вместо неё пользователю подсовывается **фейковый модуль
`matplotlib.pyplot`**, который:

-   перехватывает вызовы `plot`, `bar`, `pie`, `hist` и других функций
-   сохраняет параметры построения
-   выводит текстовое описание диаграммы через `print_ax(...)`

Поэтому:

-   тестер работает **очень быстро**
-   не требует установленного `matplotlib`
-   работает даже в средах без GUI

------------------------------------------------------------------------

# Использование

Тестер запускается как **обертка пользовательской программы** через
настройку задачи `start_cmd`.

    start_cmd = "../matplotlib_tester.py"

Тестер читает исходный код программы пользователя, добавляет в конец
вызов `print_ax(...)` и выполняет программу.

------------------------------------------------------------------------

# Настройки

Тестер настраивается через **переменные окружения**.

Обычно тестер кладут в каталог `problems` и используют **один тестер для
всех задач контеста**.

Пример:

    start_env = "PRINT_LINES=1"
    start_env = "PRINT_BARS=1"

------------------------------------------------------------------------

# Полный список параметров

## Геометрия графика

**PRINT_LINES** --- `0/1`\
печать линий и их точек (`plt.plot`)

**PRINT_BARS** --- `0/1`\
печать прямоугольников (`plt.bar` и `plt.hist`)

**PRINT_PIES** --- `0/1`\
печать секторов круговой диаграммы (`plt.pie`)

------------------------------------------------------------------------

## Подписи

**PRINT_PIE_LABELS** --- `0/1`\
печать подписей секторов pie и их координат

**PRINT_XTICKS** --- `0/1`\
печать подписей по оси X

**PRINT_TITLE** --- `0/1`\
печать заголовка графика

**PRINT_AXIS** --- `0/1`\
печать подписей осей X и Y

**PRINT_LABELS** --- `0/1`\
печать `label=` у линий

------------------------------------------------------------------------

## Проверка подписей pie

**PIE_LABELS**

Строка вида

    PIE_LABELS="A,B,C"

Тестер проверяет, что все указанные подписи присутствуют в `plt.pie`.

Если какая-то подпись отсутствует, выводится сообщение:

    ❌ PIE_LABELS: отсутствуют подписи: 'A'

------------------------------------------------------------------------

# Поддерживаемые команды matplotlib

Тестер корректно перехватывает следующие функции:

    plt.plot
    plt.bar
    plt.hist
    plt.pie
    plt.title
    plt.xlabel
    plt.ylabel
    plt.legend
    plt.xticks
    plt.subplots
    plt.gca
    plt.gcf

    ax.pie
    ax.hist
    ax.set_title
    ax.set_xlabel
    ax.set_ylabel
    ax.set_xticklabels
    ax.legend

------------------------------------------------------------------------

# Неизвестные команды

Любые **другие функции matplotlib** не вызывают ошибку.

Например:

    plt.scatter(...)
    plt.imshow(...)
    plt.fill(...)

будут **просто проигнорированы**.

Это сделано специально, чтобы тестер **никогда не падал на неизвестных
командах**.

------------------------------------------------------------------------

# Пример программы

``` python
import matplotlib.pyplot as plt

values = [40, 30, 20, 10]
labels = ['Python', 'C++', 'Java', 'Other']

plt.pie(values, labels=labels)
plt.show()
```

# Ограничения пользовательской программы

Программа должна импортировать `pyplot` под именем `plt`.

``` python
import matplotlib.pyplot as plt
```

и использовать функции `plt`:

    plt.plot(...)
    plt.bar(...)
    plt.pie(...)
    plt.hist(...)
    ...
    plt.show()

------------------------------------------------------------------------

# Использование в Polygon

Так как тестер **не требует установленного matplotlib**, его можно
использовать для генерации тестов в средах, где библиотека недоступна
(например, в Polygon).

# Рекомендации по созданию задач

### 1. Сравнивайте текстовый вывод

Самый простой способ проверки:

-   написать **эталонное решение**
-   сравнивать вывод с пользовательским

### 2. Проверяйте только нужные параметры

Например:

    PRINT_LINES=1

если важны только координаты линии.

### 3. Используйте PIE_LABELS

    PIE_LABELS="Python,C++,Java"

для проверки содержимого диаграммы.

### 4. Histogram автоматически проверяется как bar

    plt.hist(...)

превращается в набор прямоугольников.
